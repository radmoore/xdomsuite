XDOM

Class for creating a proteome-like object from a xdom file. Provides methods for iteration and filtering, sequence association collapsing repeats, retreiving statistics etc

Usage

 require 'xdomsuite'

 xdom = XDOM.new(ARGV[0], 0.001)
 puts xdom.res_coverage
 puts xdom.prot_coverage

 while(xdom.has_next)
  p = xdom.next_prot
  next unless p.has_domains?
  p.domains.each {|d| puts d.did}
 end

attach_fasta(fastafile)

Provides an interface to attach a FASTA file to the Proteins in self

Requires that the IDs of the Proteins in self correspond to the IDs in the FASTA file. IDs are matched using the REGEX described in FASTAHEAD

If a given ID in FASTA can be found in self, the sequence in FASTA is set to the corresponding protein using the sequence attribute and, for each domain present in protein, the domain sequence is set using domains sequence attribute

domains(did = nil, type=nil)

Returns an array of domain objects

  • If did is specified, an array of instances of did is returned, or an empty array if non were found

  • If did AND type are specified, method behaves as if only did was specified

find_uniq_arr_by_dom(did, collapse=true)

Returns an array of Proteins that contain did. If multiple proteins exist that contain a domain with the ID did, then only one (representative) Protein is returned. If collapse is true, proteins are collapsed before determining their uniqueness.

Returns nil if no protein with domain did is present in self

has_arr?(arrstr, sep=';')

Returns true if self contains a protein with the arrangement arrstr

The arrstr is a string representation of the domains in a protein, where sep signifies the character used to seperate the domain IDs.

Example: 

 "domainA;domainB;domainC", where sep = ';' (default)
